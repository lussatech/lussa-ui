Grid System

Grid system merupakan teknik layouting pada tampilan web yang sudah cukup lama ada, dan dengan segala tetek bengeknya masih bisa hidup sampai saat ini :D

Intro..

Teknik layouting web dengan metode grid jadi terkenal gara-gara 960.gs, doi pertama kali mengenalkan teknik layouting dengan membagi layout dengan segmen-segmen kolom dengan maksimal lebar 960 px, kata doi sih 960 itu golden viewport web.

Grid System pun berevolusi dengan kebutuhan tampilan untuk perangkat dengan ukuran beda-beda, sehingga golden ratio 960px diganti dengan istilah device breakpoint, buat desktop 1200, mobile 520 dan tablet 720. Lussa-UI menggunakan grid buatan nathan smith namanya unsemantic dengan beberapa tambahan untuk support di fluid-container. Dokumentasi grid ini kebanyakan diambil dari website dia, jadi kalian bisa baca lebih lanjut di unsemantic.com

Grid System lussa-ui tidak seperti grid system lainnya yang membagi layout dengan 16-12 kolom, kita menggunakan persentase ukuran kolom relatif terhadap containernya dengan kelipatan 5 dan kelipatan 33.33. Hal ini menyebabkan kode pada grid lebih mudah dibaca oleh awam sekalipun.

Container

Semua unit grid-class di tampung oleh sebuat top-level element, yaitu container. Container sendiri ada 2 tipe class

grid-container
Grid dengan ukuran maksimum 1200px (desktop breakpoint), yang secara otomatis berada pada tengah-tengah layout (center alignment)
grid-fluid
sesuai namanya, grid-fluid memiliki sifat mirip fluida (cairan) yang ukurannya selalu menyesuaikan penampungnya, grid-fluid akan selalu memiliki lebar sama dengan parent-nya.

Grid Classess

Jadi ada grid dengan nama kelas grid-x dimana "x" merupakan angka yang merepresentasikan presentase lebar grid unit relatif terhadap container. X sendiri memiliki rentang nilai dengan faktorisasi 5 dan maksimum nilainya 100 (grid-5, grid-10 .... grid-95, grid-100), ditambah juga dengan tambahan pembagi 3, yaitu grid-33 dan grid-66.

Tiap unit grid sendiri memiliki padding di sebelah kiri dan kanan, istilahnya adalah grid-gutter, grid-gutter secara default memiliki ukuran 20px. ukuran padding untuk sebelah kiri dan kanan adalah 1/2 dari grid-gutter. Grid-gutter dapat diganti pada file variabel pada variable.less.

Nested Grid, Zero Padding

Kalo kamu mau naro grid di dalam grid, kamu butuh container khusus yaitu grid-parent yang bakal menampung grid-unit didalamnya. Grid-parent akan menghapus grid-gutter membuat grid-unit yang menjadi parent container tidak memiliki padding pada kanan dan kirinya. Teknik ini juga relevan dipake untuk sekedar mengapus gutter pada grid-unit yang kamu inginkan.

Clearfix

Clearfix adalah salah satu kelemahan dari layout dengan teknik grid, apabila ada element yang tidak mengandung class grid-x didalam container, element tersebut akan saling meniban dengan grid dan berbagai keanehan lainnya. Hal ini karena grid menggunakan teknik floating. Apabila kamu mau menambahkan element non grid kayak span atau div, kamu perlu mengatur clearfix secara manual atau bisa menggunakan class .clear pada element tersebut

Source class clearfix pada element grid yang bisa kamu gunakan

{% highlight css %} .clearfix:before, .clearfix:after { content: "."; display: block; overflow: hidden; visibility: hidden; font-size: 0; line-height: 0; width: 0; height: 0; } .clearfix:after { clear: both; } /* ie 7 */ .clearfix { zoom: 1; } {% endhighlight %}

PS: Kedepannya kita ga akan menggunakan grid dengan semua masalah-masalah klasiknya, kita akan mengganti-nya dengan CSS3 flexBox
