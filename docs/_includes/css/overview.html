Overview

Beberapa teknologi baru & aturan-aturan dasar yang digunakan untuk menggantikan standar default browser sebelum menggunakan lussa ui

HTML5 Doctype

lussa ui menggunakan beberapa tambahan direktif untuk html 5, seperti web-component dan beberapa modul pada angular, anda bisa menggunakan html5 boilerplate, atau bisa secara manual menambahkan doctype html5

Mobile First

lussa ui dibuat pada mindset awal ditujukan untuk tampilan mobile di-awal. Tiap kode pada komponen-komponen lussa UI di utamakan untuk tampilan mobile. Untuk mendukung fungsi ini, kamu harus mendefiniskan viewport meta tag pada <head>

{% highlight html %} {% endhighlight %}

Dengan tambahan meta ini user tidak dapat melakukan fungsi pinc-to-zoom, masalah ukuran huruf dan component tidak menjadi masalah karena lussa-ui dibangun dengan ukuran dan bentuk yang responsif menyesuaikan dengan ukuran layar perangkat.

Drawback untuk metode ini bisa dilakukan menggunakan javascript (god i hate polyfill), saran saya bisa pake http://adactio.com/journal/4470/

Tapi secara default script tadi ga kita include, jadi bisa kamu tambahin sendiri, kalo mau support browser-browser tua kayak IE

Modular dan Mudah di Custom

Karena lussa-ui menggunakan angular dan CSS-Preprocessor yaitu less-css. Ukuran huruf, font serta icon yang digunakan bisa dengan mudah diganti sesuai kebutuhan, dengan mengubah file variable, font, icon atau theme di masing-masing file.

Normalize.css

Agar mendukung cross-browser rendering, lussa-ui perlu melakukan fungsi normalisasi komponen-komponen web yang biasa-nya beda-beda untuk tiap browser. Disini kita pake Normalize.css, punya Nicolas Gallagher dan Jonathan Neal yang sudah PoC dan banyak digunakan khalayak ramai.
