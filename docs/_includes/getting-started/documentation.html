Dokumentasi

dokumentasi di build menggunakan static web generator yang biasa digunakan oleh github yaitu jekyll

Requirement

Sebelum dapat melakukan mengombpile documentation, pastikan sistem di komputer kamu sudah memiliki libary / runtime berikut

  • Ruby, termasuk header development library-nya
  • RubyGems
  • Rouge
  • Jekyll

Instalasi Ruby

Untuk installasi ruby, disarankan menggunakan RVM (ruby version manager) untuk linux, sedangkan untuk windows kamu bisa menggunakan rubyinstaller.

Instalasi Header Development Ruby

Untuk windows bisa dilihat di sini.

Instalasi RubyGems

Untuk linux bisa dilihat disini. Untuk windows secara otomatis sudah terinstall jika menggunakan ruby installer.

Instalasi Jekyll

Jekyll dapat dengan mudah diinstall menggunakan rubygem

{% highlight shell %} gem install jekyll {% endhighlight %}

Konfirmasi instalasi dengan menggunakan command ini

{% highlight shell %} jekyll -v {% endhighlight %}

Instalasi Rouge

By default, jekyll menggunakan pygments untuk merender syntax higlighting, tetapi untuk OS windows, pygments terkadang sering tidak berjalan, dan perlu menginstall python untuk membuatnya dapat berjalan. Alternatifnya kamu bisa pake rouge yang ditulis menggunakan ruby.

{% highlight shell %} gem install rouge {% endhighlight %}

Build the Documentation

Untuk sekedar build saja, kamu cukup mengeksekusi command ini

{% highlight shell %} grunt docs {% endhighlight %}

File dokumentasi ada di docs/_site/

Jika ingin sekalian menjalankan http server, pakai command ini

{% highlight shell %} grunt docs-deploy {% endhighlight %}

Buka localhost:4190

Pengujian

Kalau kamu seandainya ngulik-ngulik code Lussa-UI kamu bisa nge-test ulikan kamu dengan menggunakan command ini

{% highlight shell %} grunt test # atau bisa juga via NPM script npm test {% endhighlight %}
